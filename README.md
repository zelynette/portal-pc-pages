# portal-pc-pages

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库本身不绑定任何特定域名或网站，主要作为页面资源的集中存放与版本管理用途。

## 目录结构

```
portal-pc-pages/
├── pages/          # 存放所有 HTML 页面文件
│   ├── example/    # 示例页面目录（可替换）
│   └── ...
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── docs/           # 文档或辅助说明
└── README.md       # 本文件
```

## 页面归档说明

- 所有 HTML 页面均放置在 `pages/` 目录下，每个页面可拥有独立的子目录。
- 页面资源（样式、脚本、图片等）建议集中存放于 `assets/`，便于统一管理。
- 页面之间相互独立，不依赖仓库外的其他资源。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 页面。
3. 提交变更并推送至远程仓库。
4. 可通过 GitHub Pages 或其他静态托管服务部署页面内容。

## 维护说明

- 本仓库由维护者定期更新，添加或修正页面内容。
- 欢迎提交 Issue 或 Pull Request 以改进页面质量或修复问题。
- 仓库不承诺提供任何形式的在线服务或内容更新频率。

## 贡献指南

- 新增页面时，请在 `pages/` 下创建有意义的目录名称。
- 保持资源文件路径的相对性，避免使用绝对路径。
- 提交前请确保 HTML 页面在本地可正常打开。
